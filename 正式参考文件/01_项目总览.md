# 光伏智能问答系统第一阶段项目总览

## 1. 项目背景
本项目面向光伏运维场景，目标是构建一个可用于文档问答的智能助手。第一阶段优先围绕现有文档库和光伏运维故障案例库，建立一个稳定、可信、可追溯的 Web MVP，包含面向最终用户的极简问答界面，以及面向管理员的文档管理与系统调试后台。

## 2. 第一阶段目标
第一阶段不追求全能 AI 助手，而是聚焦文档知识库 RAG 闭环：
- 文档可上传并进入知识库
- 文档可解析、切块、向量化、入库
- 用户可在 Web 页面发起问答
- 系统可返回基于证据的答案
- 答案必须带来源引用
- 无依据时必须拒答或使用固定兜底话术

## 3. 第一阶段范围
### 包含内容
- 文档上传
- 文档去重
- 文档解析
- 文本清洗
- 文本切块
- 向量化与 pgvector 入库
- FAQ 优先匹配
- 文档检索与混合排序
- 基于证据的回答生成
- 来源引用展示
- 会话记录
- 未命中问题记录
- FAQ 维护与未命中问题回流闭环
- 基础日志追踪
- 登录页、用户问答页、管理员问答页、文档管理页、未命中问题页、基础日志页

### 不包含内容
- 移动端页面
- 语音输入
- 实时数据库查询
- 实时 API 调用
- SCADA / DCS 接入
- 工单系统对接
- 推理 Agent
- 时序建模
- 主动预警
- 全量 OCR 难文档治理
- 复杂扫描 PDF、接线图、图片表格精细解析
- 多租户隔离
- 完整后台运营平台

## 4. 项目路线
- 阶段一：文档知识库 RAG
- 阶段二：工具融合 Function Calling
- 阶段三：推理智能体 Reasoning Agent

## 5. 第一阶段成功标志
- 管理员可导入文档
- 文档可正确入库
- 用户可在 Web 发问
- 系统可返回带引用的答案
- 无依据时系统可拒答
- 日志和未命中问题可追踪

## 5.1 迭代闭环要求
项目周期内需预留人工审核时间，将已解决的未命中问题回流至 `faq_item` 表，或补充到知识库文档中，避免未命中问题记录成为数据孤岛，并让系统具备知识增长能力。

## 6. 结论
第一阶段的核心不是做“大而全”的 AI，而是做一个可验证、可演示、可迭代的文档问答底座。

# 项目目录结构说明

本文档描述当前初始化后的项目目录，用于两人协作时快速判断文件职责和后续模块应放在哪里。

## 根目录

```text
operation_project/
├─ backend/                 后端 FastAPI 服务
├─ frontend/                前端 Vue 3 + Vite 应用
├─ docs/                    项目协作文档与工程说明
├─ scripts/                 初始化、运维或辅助脚本
├─ 正式参考文件/             产品、架构、接口、协作等正式参考文档
├─ 信息碎片/                 早期材料、截图、草稿和原始需求碎片
├─ .env.example             环境变量示例
├─ .gitignore               Git 忽略规则
├─ docker-compose.yml       本地 PostgreSQL + Redis 容器方案
└─ README.md                项目启动入口说明
```

## backend

```text
backend/
├─ app/
│  ├─ api/                  HTTP 路由入口，后续放 documents、qa、sessions 等接口
│  ├─ core/                 配置、日志、通用基础设施
│  ├─ db/                   数据库连接、session、迁移相关辅助
│  ├─ models/               SQLAlchemy 模型，后续承载核心表结构
│  ├─ schemas/              Pydantic 请求和响应结构
│  ├─ services/             业务服务，如文档管理、检索、问答编排
│  └─ main.py               FastAPI 应用入口
├─ requirements.txt         后端运行依赖
└─ requirements-dev.txt     后端开发和测试依赖
```

后续模块建议按正式参考文件推进：

- Module 03 在 `models/` 和 `schemas/` 中补核心数据模型。
- Module 04 在 `api/`、`services/` 中补文档上传与管理。
- Module 05 增加解析器、清洗器、切块器和入库服务。
- Module 06 增加 FAQ 优先匹配、混合检索、回答生成和引用组装。
- Module 07 增加会话、日志、未命中问题相关接口和服务。

## frontend

```text
frontend/
├─ src/
│  ├─ router/               前端路由
│  ├─ styles/               全局样式
│  ├─ views/                页面级组件
│  ├─ App.vue               应用根组件
│  ├─ main.ts               前端入口
│  └─ vite-env.d.ts         Vite 类型声明
├─ index.html               HTML 入口
├─ package.json             前端依赖和脚本
├─ tsconfig.json            TypeScript 配置
├─ tsconfig.node.json       Vite 配置类型环境
└─ vite.config.ts           Vite 开发服务器和代理配置
```

后续页面建议按 `05_前端页面设计.md` 扩展：

- 登录页
- 问答页
- 文档管理页
- 未命中问题页
- 基础日志页

通用组件建议放在 `frontend/src/components/`，接口调用建议放在 `frontend/src/api/`，类型定义建议放在 `frontend/src/types/`。这些目录会在对应模块开始时创建，避免当前初始化阶段提前放空目录。

## scripts

```text
scripts/
└─ init-db.sql              Docker PostgreSQL 初始化扩展脚本
```

当前脚本用于启用 `vector` 和 `pg_trgm` 扩展。后续如增加本地数据导入、评测或演示脚本，也统一放在该目录。

## docs

```text
docs/
└─ 项目目录结构说明.md       当前文件
```

后续建议逐步补充：

- `api-contract.md`：接口契约
- `database-schema.md`：数据库结构
- `progress.md`：模块推进记录
- `acceptance.md`：验收记录和演示脚本

## 当前初始化边界

当前目录只完成 Module 01 项目底座，不包含真实业务闭环。已经准备好的能力是：

- 后端 FastAPI 应用入口和健康检查
- 前端 Vue 3 + Vite 应用入口和占位首页
- PostgreSQL + pgvector + Redis 的 Docker Compose 示例
- 环境变量示例
- 后续模块目录约定

下一步建议进入 Module 02 前端页面同步实现，或先补 Module 03 API 契约与数据模型，具体顺序可按团队当前分工决定。
